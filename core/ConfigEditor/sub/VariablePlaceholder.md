## 変数プレースホルダー

### 概要

- **目的**: 変数の概念を導入し、より複雑で自由度の高いおみくじ結果を表示できるようにします。
- **特徴**: 簡易的なプログラミング要素を含みます。予約語以外は変数を保存できるため、簡単な TRPG、占いランキングを実装できます。
	- `expr-eval-fork` ライブラリを使用しています。
- **チートシート**: [VariablePlaceholderCheatSheet](./VariablePlaceholderCheatSheet.md)

### 利用可能な場所

以下の設定項目内で使用できます。

- **各アクションセット** ＞ **おみくじ設定**
    - フキダシ内容
    - WordParty
- **プレースホルダー**

### 基本的な書式

使用したい機能を `{{ ... }}` のように**波括弧（`{}`）で二重に囲んで記述します**。

例: `{{score = 100}}`, `{{score += 10}}`, `{{score}}`

---

### 機能一覧と詳細

#### 1. 変数への代入

**値を変数に保存します。**

- **書式**: `{{変数名 = 値}}`
- **変数名のルール**: 英数字が使用できます。先頭の文字は**英字**にしてください。
- 一度代入した変数は、**ブラウザ（または OBS）を閉じるまで保持され続けます**。

```javascript
{{name = 'さくら'}}     // 文字列（シングルクォート必須）
{{score = 100}}          // 数値
{{copy = score}}         // 別の変数の値をコピー（未定義なら 0）
```

#### 2. 変数参照（出力）

**保存した変数の値をテキストとして出力します。**

- **書式**: `{{変数名}}`
- 代入タグ（`=` を含むもの）はテキストに何も出力しません。値を表示したい場合はこの参照タグを使います。

```javascript
{{score = 100}}
あなたのスコアは {{score}} 点です！
// → あなたのスコアは 100 点です！
```

#### 3. 四則演算による再代入

**変数の値を使って再計算し、改めて代入します。**

- **書式**: `{{変数名 = 変数名 演算子 値}}`
- 使用可能演算子: `+`, `-`, `*`, `/`
- ※ `+=`, `-=`, `*=`, `/=` は使用できません
- **数値の範囲**: -999,999,999 ～ 999,999,999（超過時は上限/下限に丸め）
- 小数点以下は **0.01** まで有効

```javascript
{{score = score + 10}}      // score に 10 を加算
{{hp = hp - 1}}             // hp から 1 を減算
{{attack = attack * 2}}     // attack を 2 倍
{{total = total / 2}}       // total を 2 で割る
{{point = point + rand(1, 5)}}  // rand と組み合わせも可能
```

#### 4. 三項演算子（条件分岐）

**条件に応じて異なる値を代入します。**

- **書式**: `{{変数名 = 条件式 ? 真の値 : 偽の値}}`
- 使用できる比較演算子: `==`, `!=`, `>`, `<`, `>=`, `<=`
- 数値の比較のみ使用可能です（文字列比較は不可）

```javascript
{{score += 10}}
{{result = score >= 50 ? 'クリア' : 'ゲームオーバー'}}
[[result]]
// score が 50 以上なら「クリア」、未満なら「ゲームオーバー」
```

#### 5. 複数式の同時実行

**セミコロン（`;`）区切りで複数の式を 1 つのタグに書けます。**

```javascript
{{score = 0; hp = 100; name = 'hero'}}
```

#### 6. 型変換・文字列操作関数

式内では暗黙の型変換は行われません。必要に応じて明示的な変換関数を使用します。

**`str()` — 文字列化**

- 数値や変数を安全に文字列へ変換します
- 文字列連結は必ず `str()` を使用します

```javascript
{{text = str('1/', score)}}
```

**`num()` — 数値化**

- 値を数値へ変換します
- 変換できない場合は `0` を返します

```javascript
{{value = num(textValue)}}
```

---

**`bool()` — 真偽値化（1 / 0）**

- `0` を `0`
- それ以外を `1` に変換します

```javascript
{{flag = bool(score)}}
{{result = bool(score) ? 'OK' : 'NG'}}
```

---

#### 7. `rand` - ランダムな数値の生成

**指定した範囲内のランダムな整数を生成します。**

- **書式**: `{{変数名 = rand(最小値, 最大値)}}`
- 複合代入とも組み合わせ可能

```javascript
{{dice = rand(1, 6)}}
{{score += rand(1, 10)}}
[[dice]]  // 結果を出力
```

### 応用的な使い方とルール

#### 値の代入ルール

`{{変数名 = 右辺}}` を記述したとき、右辺は以下のルールで解釈されます。

|記述例|解釈|結果|
|---|---|---|
|`{{a = 'hello'}}`|シングルクォートあり|文字列 `"hello"`|
|`{{a = 100}}`|数値のみ|数値 `100`|
|`{{a = other}}`|変数名|変数 `other` の値（未定義なら `0`）|
|`{{a = score >= 10 ? 1 : 0}}`|`?` を含む|三項演算子として式評価|

**文字列を保存したい場合は必ずシングルクォートで囲んでください。**

#### 処理の順序

1. **プレースホルダー → 変数プレースホルダー**の順に処理されます。
    - 例: `{{userId = <<ユーザーID>>}}` → `<<ユーザーID>>` が置き換えられてから変数に格納
2. 変数名にもプレースホルダーを使用できます。
    - 例: `{{<<アイテム名>> = <<価格>>}}`

#### 予約語（特別な変数）

以下の変数名はシステムが特別な意味を持つ**予約語**です。

- **予約語の一例**: `text`, `symbol`, `name`, `variant`, `unique`, `slot0` ～ `slot9`
- 各予約語の役割は連携するアイテムによって異なります。

### 旧構文について（非推奨、v3 で廃止）

以下の構文は旧バージョンとの互換性のため引き続き動作しますが、新しく書く場合は上記の新構文を推奨します。

| 旧構文（非推奨）                 | 新構文（推奨）                          |
| ------------------------ | -------------------------------- |
| `{{var name = 'value'}}` | `{{name = 'value'}}`             |
| `{{rand(1, 10)}}`        | `{{dice = rand(1, 10)}}[[dice]]` |
